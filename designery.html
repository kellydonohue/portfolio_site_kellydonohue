designery

centralizes the furniture shopping experience to one native app. Find inspiration, compare options from different stores, and buy everything you need, all in one place. This project was completed as part of CareerFoundry’s UI Design course.

my role

As someone who is passionate about designing my own space but struggles to find affordable prices and cohesive pieces, I saw an opportunity to simplify the furniture shopping process. I was responsible for the User Interface Design for this project, including market research, user flows, wireframes, and prototyping for iOS and Android.

research

I started this process by conducting market research on existing furniture shopping apps. While some stores have great apps or websites for their own products, there is no easy way to compare options between stores.

that's where designery comes in.

I then researched existing universal shopping carts, which allow users to buy items from multiple stores all in one place. The technology is there, but there are none that focus specifically on interior design.

get inspired

To define user flows, I placed an emphasis on inspiration. The discover option lets users browse curated rooms by interior design style. This shows how unique pieces look together, and visualizes a cohesive space.

streamlined search

For users who know what they’re looking for, the search function lets users browse all their options, filtering by price, color, or brand. They can add directly to the shopping cart or save favorite items for later.

I started the process of converting user flows into wireframes by defining an overall aesthetic and feeling I want to convey through this design.

sleek. minimal. neutral.

The products for sale should be the main focus of this app; the design should not distract or take away from them. I kept this in the back of my head as I start to gain pattern inspiration and research the iOS and Material Guidelines.

I then created low-fidelity wireframes for each step in my user flow for both iOS and Android devices.

The main differences between the iOS and Android versions of this app are the navigational structure and the search function. The iOS version uses a bottom navigation bar, while the Android version uses a hamburger menu.

why?

sleek, minimal, neutral. Android phones have a built in bottom navigation bar. Adding another bar on top of it creates busyness, which I want to avoid.

After applying styling to all screens, I tested it with four users to get feedback on intuitiveness and aesthetic of the app.

“The overall design looks very clean, minimalistic and sophisticated.”

The visual design style I aimed for was being communicated.

“The account page looks a little busy with text, and the menu looks really empty.”

Based on this constructive feedback, I decided to rethink the account page. This user is referring to the hamburger menu, which is only on Android, but the account page layout is consistent across both platforms.

I moved the settings and help & info tabs to the main menu for easier access, which opened up space on the account page.

Recent orders are now shown as cards, to reduce text and add a visual element to the screen.
